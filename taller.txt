



Taller 15: Desarrollo de Aplicación Fitness con Platform Channels en Flutter




1. Introducción


1.1 Objetivo del Taller


Construir una aplicación de fitness que utiliza Platform Channels para acceder a funcionalidades nativas de Android como: 


- Autenticación biométrica (huella dactilar) 


- Contador de pasos (acelerómetro) 


- Tracking de rutas GPS





1.2¿Qué son los Platform Channels?


1.2.1Concepto Fundamental


Los Platform Channels son el puente de comunicación entre el código Flutter (Dart) y el código nativo (Kotlin/Java en Android, Swift/Objective-C en iOS).





1.2.2Tipos de Platform Channels


MethodChannel - Comunicación Petición/Respuesta


// Flutter llama → Android respondefinal result = await methodChannel.invokeMethod('metodoDePrueba');


Casos de uso: 


- Verificar disponibilidad de biometría 


- Solicitar permisos 


- Obtener ubicación actual


EventChannel - Streaming de Datos


// Android envía datos continuamente → Flutter escuchaeventChannel.receiveBroadcastStream().listen((data) {  print('Nuevo dato: $data');});


Casos de uso: 


- Stream de datos del acelerómetro 


- Actualizaciones continuas de GPS 


- Monitoreo de sensores

package com.tuinstituto.fitness_trackerimport android.os.Bundleimport androidx.biometric.BiometricManagerimport androidx.biometric.BiometricPromptimport androidx.core.content.ContextCompatimport io.flutter.embedding.android.FlutterFragmentActivityimport io.flutter.embedding.engine.FlutterEngineimport io.flutter.plugin.common.MethodChannelimport java.util.concurrent.Executor/** * MainActivity: punto de entrada de la aplicación Android * - Extiende FlutterFragmentActivity (necesario para BiometricPrompt) * - Configura los Platform Channels aquí */class MainActivity: FlutterFragmentActivity() {    // PASO 1: Definir nombre del canal (DEBE coincidir con Dart)    private val BIOMETRIC_CHANNEL = "com.tuinstituto.fitness/biometric"    // PASO 2: Variables para biometría    private lateinit var executor: Executor    private lateinit var biometricPrompt: BiometricPrompt    private var pendingResult: MethodChannel.Result? = null    /**     * configureFlutterEngine: se llama al iniciar la app     * AQUÍ configuramos TODOS los Platform Channels     */    override fun configureFlutterEngine(flutterEngine: FlutterEngine) {        super.configureFlutterEngine(flutterEngine)        // Inicializar executor para biometría        executor = ContextCompat.getMainExecutor(this)        // CONFIGURAR PLATFORM CHANNEL - BIOMETRÍA        MethodChannel(            flutterEngine.dartExecutor.binaryMessenger,            BIOMETRIC_CHANNEL        ).setMethodCallHandler { call, result -&gt;            /**             * setMethodCallHandler: escucha llamadas desde Flutter             *             * Parámetros:             * - call: contiene el nombre del método y argumentos             * - result: objeto para enviar respuesta a Flutter             */            when (call.method) {                "checkBiometricSupport" -&gt; true            else -&gt; false        }    }    /**     * Mostrar diálogo de autenticación biométrica     */    private fun showBiometricPrompt() {        // Configurar información del diálogo        val promptInfo = BiometricPrompt.PromptInfo.Builder()            .setTitle("Autenticación Biométrica")            .setSubtitle("Usa tu huella dactilar")            .setDescription("Coloca tu dedo en el sensor")            .setNegativeButtonText("Cancelar")            .setAllowedAuthenticators(BiometricManager.Authenticators.BIOMETRIC_STRONG)            .build()        // Crear BiometricPrompt con callbacks        biometricPrompt = BiometricPrompt(this, executor,            object : BiometricPrompt.AuthenticationCallback() {                override fun onAuthenticationSucceeded(                    result: BiometricPrompt.AuthenticationResult                ) {                    super.onAuthenticationSucceeded(result)                    //  Autenticación exitosa                    pendingResult?.success(true)                    pendingResult = null                }                override fun onAuthenticationError(                    errorCode: Int,                    errString: CharSequence                ) {                    super.onAuthenticationError(errorCode, errString)                    // ❌ Error en autenticación                    pendingResult?.success(false)                    pendingResult = null                }                override fun onAuthenticationFailed() {                    super.onAuthenticationFailed()                    // Usuario puede reintentar                }            }        )        // Mostrar el diálogo        biometricPrompt.authenticate(promptInfo)    }}

// Importaciones adicionalesimport android.hardware.Sensorimport android.hardware.SensorEventimport android.hardware.SensorEventListenerimport android.hardware.SensorManagerimport kotlin.math.sqrtimport io.flutter.plugin.common.EventChannel// Dentro de MainActivity, agregar:private val ACCELEROMETER_CHANNEL = "com.tuinstituto.fitness/accelerometer"// En configureFlutterEngine(), agregar después de la biometría:setupAccelerometerChannel(flutterEngine)// Al final de la clase, agregar método:/** * Configurar EventChannel para acelerómetro * * EXPLICACIÓN DIDÁCTICA: * - EventChannel.StreamHandler tiene 2 métodos: *   1. onListen: cuando Flutter comienza a escuchar *   2. onCancel: cuando Flutter deja de escuchar */private fun setupAccelerometerChannel(flutterEngine: FlutterEngine) {    val sensorManager = getSystemService(SENSOR_SERVICE) as SensorManager    val accelerometer = sensorManager.getDefaultSensor(Sensor.TYPE_ACCELEROMETER)    var stepCount = 0    var lastMagnitude = 0.0    var sensorEventListener: SensorEventListener? = null    // Variables para suavizado    val magnitudeHistory = mutableListOf&lt;Double&gt;()    val historySize = 10    var sampleCount = 0    var lastActivityType = "stationary"    var activityConfidence = 0    // ═══════════════════════════════════════════════════════════    // CONFIGURAR EVENT CHANNEL    // ═══════════════════════════════════════════════════════════    EventChannel(        flutterEngine.dartExecutor.binaryMessenger,        ACCELEROMETER_CHANNEL    ).setStreamHandler(object : EventChannel.StreamHandler {        /**         * onListen: Flutter comenzó a escuchar el stream         * AQUÍ iniciamos el sensor         */        override fun onListen(arguments: Any?, events: EventChannel.EventSink?) {            sensorEventListener = object : SensorEventListener {                override fun onSensorChanged(event: SensorEvent?) {                    event?.let {                        // Calcular magnitud del vector                        val x = it.values[0]                        val y = it.values[1]                        val z = it.values[2]                        val magnitude = sqrt((x * x + y * y + z * z).toDouble())                        // Promedio móvil para suavizar                        magnitudeHistory.add(magnitude)                        if (magnitudeHistory.size &gt; historySize) {                            magnitudeHistory.removeAt(0)                        }                        val avgMagnitude = magnitudeHistory.average()                        // Detectar paso                        if (magnitude &gt; 12 &amp;&amp; lastMagnitude &lt;= 12) {                            stepCount++                        }                        lastMagnitude = magnitude                        // Determinar actividad (con promedio)                        val newActivityType = when {                            avgMagnitude &lt; 10.5 -&gt; "stationary"                            avgMagnitude &lt; 13.5 -&gt; "walking"                            else -&gt; "running"                        }                        // Solo cambiar si hay confianza                        if (newActivityType == lastActivityType) {                            activityConfidence++                        } else {                            activityConfidence = 0                        }                        val finalActivityType = if (activityConfidence &gt;= 3) {                            newActivityType                        } else {                            lastActivityType                        }                        lastActivityType = newActivityType                        // Enviar cada 3 muestras                        sampleCount++                        if (sampleCount &gt;= 3) {                            sampleCount = 0                            // ENVIAR DATOS A FLUTTER                            val data = mapOf(                                "stepCount" to stepCount,                                "activityType" to finalActivityType,                                "magnitude" to avgMagnitude                            )                            // events?.success: envía datos al stream                            events?.success(data)                        }                    }                }                override fun onAccuracyChanged(sensor: Sensor?, accuracy: Int) {}            }            // Registrar listener del sensor            sensorManager.registerListener(                sensorEventListener,                accelerometer,                SensorManager.SENSOR_DELAY_GAME            )        }        /**         * onCancel: Flutter dejó de escuchar         * AQUÍ detenemos el sensor         */        override fun onCancel(arguments: Any?) {            sensorEventListener?.let {                sensorManager.unregisterListener(it)            }            sensorEventListener = null        }    })    // MethodChannel auxiliar para control    MethodChannel(        flutterEngine.dartExecutor.binaryMessenger,        "$ACCELEROMETER_CHANNEL/control"    ).setMethodCallHandler { call, result -&gt;        when (call.method) {            "start" -&gt; {                stepCount = 0                result.success(null)            }            "stop" -&gt; {                result.success(null)            }            "reset" -&gt; {                stepCount = 0                result.success(null)            }            else -&gt; result.notImplemented()        }    }}

MEGA RETO: Detector de actividad física con aviso por voz

En este reto van a implementar que su aplicación detecte en tiempo real si el usuario está caminando, corriendo o se ha caído, y que comunique cada cambio de estado mediante síntesis de voz. No se trata solo de que el código funcione: también tienen que justificar las decisiones técnicas que tomaron. Ya no es necesario usar portchannels sino plugins.

1. Funcionalidad esperada

La aplicación debe cumplir lo siguiente:

Detectar de forma continua si el usuario está caminando, corriendo o ha dejado de moverse.

Cada vez que el estado de actividad cambie, la aplicación debe anunciarlo en voz alta. Por ejemplo: "Estás corriendo" o "Cambiaste a caminata".

Detectar cuando el usuario se ha caído y mostrar inmediatamente un diálogo que le pregunte si está bien. Si el usuario no responde en 15 segundos, el diálogo debe mostrar un mensaje secundario que refuerce la solicitud de confirmación.

Evitar avisos repetitivos o falsos: el aviso de voz solo debe dispararse si el nuevo estado se ha mantenido estable durante al menos un intervalo de tiempo que el estudiante debe definir y justificar.

2. Decisiones técnicas que deben tomar

2.1 Detección de actividad

Tienen dos opciones para detectar si el usuario camina o corre: el paquete sensors_plus, que te da acceso a los datos crudos del acelerómetro, o el paquete activity_recognition_flutter, que ya clasifica la actividad. Deben analizar ambos, elegir uno y justificar su elección.

Para orientar su análisis, consideren al menos:

Qué devuelve cada paquete y que tienen que procesar ustedes de forma independiente.

Cuál tiene mejor soporte para Android

Cuál le resulta más adecuado dado el alcance de esta aplicación.

2.2 Detección de caída

Ninguno de los dos paquetes mencionados anteriormente detecta caídas directamente. Para esto necesitás trabajar con los datos crudos del acelerómetro usando sensors_plus, independientemente del paquete que hayas elegido para la actividad.

Una caída produce un cambio brusco en la aceleración del dispositivo. Para detectarla, investiguen cómo calcular la magnitud resultante de los tres ejes del acelerómetro (x, y, z) y determinen a partir de qué valor ese cambio puede interpretarse como un impacto. Ese umbral deben definirlo y justificarlo.

Consideren también que el acelerómetro es ruidoso y puede generar picos falsos. Tu solución debe contemplar algún mecanismo para reducir los falsos positivos.

2.3 Síntesis de voz

Para los avisos de voz pueden usar flutter_tts u otra alternativa que encuentren. Si eligen una opción diferente, justificar por qué consideraron que es mejor para este caso.

El aviso de voz debe sonar en el idioma que configure el sistema del dispositivo, o en español como mínimo. 

2.4 Control del ruido del sensor

Los sensores de movimiento cambian de estado con frecuencia y no siempre reflejan un cambio real de actividad. Si conectan el aviso de voz directamente al stream del sensor, la aplicación va a hablar cada pocos segundos con datos que no representan nada significativo.

Para evitar esto, deben implementar un mecanismo de debounce: el aviso de voz solo se dispara si el nuevo estado se ha mantenido estable durante un tiempo que el estudiante defina, y siempre que sea diferente al último estado que ya se anunció.

Investiguen qué es un debounce en el contexto de streams de datos, definan el tiempo que van a usar y justifiquen ese valor.

3. Permisos requeridos

La aplicación accede a sensores del dispositivo, lo que requiere permisos explícitos del sistema operativo. Deben declarar los permisos en el archivo de configuración AndroidManifest.xml y también solicitarlos al usuario en tiempo de ejecución antes de intentar acceder a cualquier sensor.

Si la aplicación omite este paso, el sensor simplemente no devuelve datos y no lanza ningún error visible. Identifiquen cuáles son los permisos específicos que necesitan.
